# 1 — Growth Hacker

> [!info] Conceito central
> **Growth Hacking** é a abordagem de crescimento que combina marketing, produto e dados para identificar e explorar alavancas de crescimento com o menor custo possível. O growth hacker não separa canal de produto — testa hipóteses rapidamente em todo o funil para encontrar o que escala. A mentalidade central: **crescer com evidências, não com opiniões**.

---

## O Funil AARRR (Pirate Metrics — Dave McClure)

| Etapa | O que é | Pergunta central |
|---|---|---|
| **Acquisition** | Como os usuários chegam | Qual canal traz usuários com menor CAC? |
| **Activation** | Primeira experiência de valor | Quantos chegam ao "aha moment"? |
| **Retention** | Usuários que voltam | Quantos continuam usando após D7/D30? |
| **Revenue** | Monetização | Quanto cada usuário gera? |
| **Referral** | Usuários que indicam outros | O produto se distribui sozinho? |

> [!tip] Onde focar primeiro
> Antes de escalar aquisição (mídia paga), corrija activation e retention. Escalar tráfego para um produto que não retém é encher balde furado — custo alto, crescimento nulo.

---

## O processo de Growth — ciclo de experimentos

```
1. ANÁLISE — Onde está o maior gargalo no funil?
2. HIPÓTESE — "Se fizermos X, Y vai melhorar por causa de Z"
3. PRIORIZAÇÃO — ICE Score: Impact × Confidence × Ease
4. EXPERIMENTO — Teste controlado (A/B ou split test)
5. APRENDIZADO — O que os dados dizem? (não o que achamos)
6. ESCALA — Se funcionou: escala. Se não: próxima hipótese
```

---

## Frameworks de priorização

### ICE Score

| Critério | Pergunta | Pontuação |
|---|---|---|
| **Impact** | Se funcionar, qual o impacto no KPI? | 1–10 |
| **Confidence** | Qual a certeza de que vai funcionar? | 1–10 |
| **Ease** | Quão fácil de implementar? | 1–10 |

**ICE = (Impact + Confidence + Ease) / 3**

### RICE Score (Intercom)

| Critério | O que mede |
|---|---|
| **Reach** | Quantas pessoas afeta |
| **Impact** | Quanto muda o resultado |
| **Confidence** | % de certeza |
| **Effort** | Horas de trabalho para implementar |

## Loops de crescimento — além do funil

Um **growth loop** é um sistema onde o output de uma etapa alimenta o input da próxima — criando crescimento composto:

| Loop | Exemplo | Empresas |
|---|---|---|
| **Viral loop** | Usuário convida amigo → amigo convida mais amigos | WhatsApp, Duolingo |
| **Content loop** | Usuário cria conteúdo → SEO atrai mais usuários → mais criadores | Pinterest, Reddit |
| **Paid loop** | Receita → reinvestimento em mídia → mais clientes → mais receita | E-commerce |
| **Product loop** | Produto melhora com uso → mais engajamento → mais dados | Spotify, Netflix |

## Growth vs Marketing Tradicional

| Dimensão | Marketing Tradicional | Growth Hacking |
|---|---|---|
| **Foco** | Campanhas e canais | Funil completo + produto |
| **Ciclo** | Trimestral / anual | Semanal / quinzenal |
| **Decisão** | Intuição + experiência | Dados + experimento |
| **Métrica** | Alcance, awareness | Receita, retenção, LTV |
| **Orçamento** | Budget fixo | Alocação por ROI |

---

## Anti-padrões frequentes

1. **Chamar de growth hacking qualquer hack de marketing.** Growth sem dados, sem ciclo de experimentos e sem conexão com o produto é só marketing renomeado.

2. **Focar só em aquisição.** O maior alavancador de crescimento é retenção. Um produto com retenção D30 de 40% cresce com qualquer canal de aquisição. Com 5%, nenhum canal resolve.

3. **Testar sem hipótese clara.** Experimento sem hipótese é aleatoriedade. Hipótese = "Se [ação], então [resultado mensurável] porque [razão]". Sem isso, não se aprende nada com o teste.

4. **Escalar experimento antes da significância estatística.** Resultado promissor com N=50 pode ser ruído. Aguardar significância (p < 0.05) antes de escalar evita decisões baseadas em acaso.

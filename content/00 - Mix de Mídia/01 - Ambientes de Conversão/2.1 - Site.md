# 2.1 — Site

> [!info] Conceito central
> O **site** é o único ambiente digital 100% sob controle da marca — sem algoritmos de terceiros, sem comissão de marketplace, sem risco de mudança de regras de plataforma. É a base da presença digital: onde tráfego de todos os canais converge, onde a marca fala com sua própria voz e onde acontece a conversão mais rentável (sem custo por transação de terceiro).

## Performance técnica — fatores críticos

### Core Web Vitals (Google)

| Métrica | O que mede | Meta |
|---|---|---|
| **LCP (Largest Contentful Paint)** | Tempo até o maior elemento visível carregar | < 2,5 segundos |
| **FID / INP (Interaction to Next Paint)** | Resposta a interações do usuário | < 200ms |
| **CLS (Cumulative Layout Shift)** | Estabilidade visual — elementos que "pulam" | < 0,1 |

> [!warning] Velocidade afeta SEO e conversão
> Google usa Core Web Vitals como fator de ranqueamento desde 2021. Além disso, cada segundo a mais de carregamento reduz conversão em ~7% (Akamai). Site lento = menos orgânico + mais bounce no tráfego pago.

## Anatomia de uma landing page de alta conversão

| Elemento | Função |
|---|---|
| **Headline** | Proposta de valor clara em < 8 palavras |
| **Sub-headline** | Detalha o benefício e para quem é |
| **Hero image / vídeo** | Reforça visualmente a proposta |
| **Social proof** | Avaliações, logos de clientes, número de usuários |
| **Benefícios (não features)** | O que o usuário ganha — não o que o produto faz |
| **CTA primário** | 1 ação clara e visível acima do fold |
| **Objeção-breaking** | FAQ, garantia, política de devolução |
| **CTA secundário** | Para quem não está pronto — capturar email |

## CRO — Conversion Rate Optimization

**CRO** é a prática sistemática de aumentar a % de visitantes que realizam a ação desejada.

### Ferramentas de CRO

| Ferramenta | Função |
|---|---|
| **Google Optimize / VWO / AB Tasty** | Testes A/B e multivariados |
| **Hotjar / Microsoft Clarity** | Heatmaps, gravações de sessão, funis |
| **GA4** | Análise de funil e abandono |
| **Crazy Egg** | Scroll maps e click maps |

### Priorização de testes (framework ICE)

| Critério | Pergunta |
|---|---|
| **Impact** | Se funcionar, quanto muda a conversão? |
| **Confidence** | Qual a certeza de que vai funcionar? |
| **Ease** | Quão fácil é implementar? |

## Anti-padrões frequentes

1. **Mesma landing page para todo o tráfego.** Usuário que veio de search brand tem intenção diferente de quem veio de prospecting no TikTok. Personalização de landing page por fonte aumenta conversão consistentemente.

2. **CTA fraco ou escondido.** Botão de cor que se mistura com o fundo, texto vago ("saiba mais"), posição abaixo do fold. CTA deve ser visível, específico e orientado ao benefício ("Começar grátis", "Ver preços").

3. **Não ter mobile-first.** No Brasil, 80%+ do tráfego é mobile. Site que só foi otimizado para desktop perde a maioria dos visitantes.

4. **Formulário com muitos campos.** Cada campo adicional reduz conversão. Pedir só o essencial (nome + email) e capturar mais dados depois via progressive profiling.
